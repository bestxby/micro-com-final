# STM32-Adaptive-Sentinel: Real-time Environmental Anomaly Monitoring & Adaptive Flappy Bird Game Engine

基于 **STM32F103C8T6** (Blue Pill) 的环境自适应异常监测与像素飞鸟小游戏系统。系统采用高内聚、低耦合的模块化设计，完美适配了 **480×320 横屏 LCD** 以及 **XPT2046 触摸屏控制器**，并支持基于电容/电阻式触屏的多点手势识别与交互。

本项目已全面升级适配 **Keil v5**，支持 **ARM Compiler 5 (armcc)** 和 **ARM Compiler 6 (armclang)** 编译环境（需启用 C99 模式）。

## 2. 软硬件开发环境

### 2.1 软件要求
* **集成开发环境 (IDE)**: Keil uVision 5
* **编译器 (Compiler)**: ARM Compiler 5 (armcc) / ARM Compiler 6 (armclang)
* **编译器设置**: 已开启 **C99 Mode** (`--c99`，在项目配置 `C/C++` 选项卡中启用)，允许在 for 循环及执行语句后定义变量。
* **包含路径 (Include Paths)**: 已配置 `.\User`、`.\RTE`、`.\Hardware`、`.\Library`
* **字符编码 (Encoding)**: 所有源文件均使用 **UTF-8** 编码，防止中文注释在不同系统上乱码。

### 2.2 默认硬件引脚接线 (实验平台: EES-351-MC)

本项目在 **EES-351-MC 微机原理实验箱** 上调试通过，STM32 排针与底板外设物理接线如下表所示：

| STM32 引脚 | 物理连接 | 功能角色 | 驱动模式 / 复用功能 |
| :--- | :--- | :--- | :--- |
| **PC13** | 板载 LED / 底板 LEDx | LED1 — 异常告警 5Hz 闪烁 | GPIO 推挽输出 |
| **PA0** | 板载 A0 排针 → LEDx | LED2 — PWM 呼吸灯 | TIM2_CH1 |
| **PA1** | 板载 A1 排针 → LEDx | LED3 — AI 学习指示灯 | GPIO 推挽输出 |
| **PA2** | 板载 A2 排针 → LEDx | LED4 — AI 监测指示灯 | GPIO 推挽输出 |
| **PB0** | 板载 D0 排针 → BTN1 | KEY1 — UI 页面翻页切换 | GPIO 上拉输入 |
| **PB8** | 板载 D7 排针 → BTN2 | KEY2 — 警报重置 / 飞鸟跳跃重启 | GPIO 上拉输入 |
| **PA3** | 板载 A3 排针 → TFT RST | LCD 硬件复位 | GPIO 推挽输出 |
| **PA4** | 板载 A4 排针 → TFT CS | LCD 片选 (SPI CS) | GPIO 推挽输出 |
| **PA5** | 板载 A5 排针 → TFT SCK | LCD SPI 时钟 (SCLK) | GPIO 推挽输出 |
| **PA7** | 板载 B4 排针 → TFT SDI | LCD SPI MOSI 数据线 | GPIO 推挽输出 |
| **PA8** | 板载 B3 排针 → TFT DC | LCD RS 命令/数据选择 | GPIO 推挽输出 |
| **PB1** | 板载 D6 排针 → TFT LED | LCD 背光控制 | GPIO 推挽输出 |
| **PB6** | 板载 D5 排针 → I²C SCL | I²C 时钟线 (模拟 AHT20/BMP280) | GPIO 模拟开漏输出 |
| **PB7** | 板载 D6 排针 → I²C SDA | I²C 数据线 (模拟 AHT20/BMP280) | GPIO 模拟开漏输出 |
| **PA0** *(复用)* | 板载 A0 排针 → TFT T_IRQ | 触摸中断引脚 (PEN) | GPIO 上拉输入 (共享 PA0) |
| **PA1** *(复用)* | 板载 A1 排针 → TFT T_DO | 触摸 SPI MISO (DOUT) | GPIO 上拉输入 (共享 PA1) |
| **PB3** | 板载 D3 排针 → TFT T_DIN | 触摸 SPI MOSI (TDIN) | GPIO 推挽输出 (解除 JTAG 占用) |
| **PA8** *(复用)* | 板载 B3 排针 → TFT T_CLK | 触摸 SPI SCK (TCLK) | GPIO 推挽输出 (共享 PA8) |
| **PB4** | 板载 D4 排针 → TFT T_CS | 触摸 SPI 片选 (TCS) | GPIO 推挽输出 (解除 JTAG 占用) |

> [!IMPORTANT]
> **PB3 (JTDO)** 与 **PB4 (JNTRST)** 默认被芯片分配给 JTAG 调试占用。为了使其能作为普通 I/O 控制触摸屏，我们在 `TP_Init` 中执行了 **SWD 接口保留，禁用 JTAG 的重映射**：
> `AFIO->MAPR = (AFIO->MAPR & ~AFIO_MAPR_SWJ_CFG) | AFIO_MAPR_SWJ_CFG_JTAGDISABLE;`

## 3. 触摸屏交互设计

### 3.1 触摸硬件层驱动 (`touch.c` & `touch.h`)
* **位带操作 (Bit-banding)**: 利用 Cortex-M3 位带区映射实现单指令快速读写 I/O 引脚（如 `TCS = 1`, `TCLK = 0`），使触摸屏软件 SPI 通信更加高速稳定。
* **低耦合设计**: 采用寄存器直接配置方式进行引脚初始化，不依赖标准外设库 `.c` 文件，避免项目文件冗余。
* **出厂预校准参数**: 针对 480×320 LCD 横屏，内置了物理 ADC 到屏幕像素坐标的映射系数：
  * `xfac = 0.135f`, `yfac = 0.090f`
  * `xoff = -40`, `yoff = -20`
  * 开机即可直接进入高灵敏度触控状态，跳过了繁琐的阻塞式校准画面。

### 3.2 触摸手势与动作检测
* **左右划屏翻页**: 支持在任意页面下左右滑动切换监测界面。
  * **向左滑动**: 切换至下一个页面 (`(current_page + 1) % 4`)；
  * **向右滑动**: 切换至上一个页面 (`(current_page + 3) % 4`)；
  * 滑动检测阈值设为 **横向移动 > 80 像素且纵向位移 < 60 像素**，有效避免对角线滑动和点击时的误触发。
* **屏幕 Tap 点击**: 在 Flappy Bird 游戏页面 (Page 3) 且未触发滑动手势时，点击游戏区域中央（`X: 20..460`, `Y: 56..286` 矩形画布内）即可触控起飞。操作体验比物理按键更佳。

## 4. 系统核心功能设计

### 4.1 传感器热插拔自愈与重连
系统每隔 1.5 秒读取一次温湿度与气压数据。若连续通信失败 3 次，系统将标记传感器为失联状态并在屏幕上高亮报错。在失联状态下，系统每个周期会自动重新初始化 I2C 和传感器，一旦重新接入传感器，系统可在 1.5 秒内实现自动重连与恢复。

### 4.2 AI 自适应基准学习与突变检测
传感器连接成功后，系统会运用 EMA 指数移动平均平滑滤波对原始读数进行平滑，并在前 100 次采样（约 150 秒）内采集数据计算拟合出环境温度的动态基线。学习结束后，若实时滤波温度偏离该基准超过 ±5°C，系统会自动判定为异常突变，并将其记录至历史日志。

### 4.3 霓虹卡片两栏式 UI 布局
* **Page 0 (监测页)**：以 Slate 暗灰色作为卡片底色，配以卡片边框。采用两栏对称布局，一屏直观展现温度、湿度、气压、AI 滤波值以及传感器状态。
* **Page 1 (自适应页)**：左侧显示 AI 动态学习温度的基线与当前偏差，右侧实时绘制 24 点示波器式温度偏差曲线折线图。
* **Page 2 (日志页)**：采用精致的霓虹时钟线与时间轴，浮动显示最近 7 条异常日志的历史快照（包括突变时刻、基准值与异常温度）。
* **Page 3 (游戏页)**：Flappy Bird 游戏主页。得分看板 (`SCORE: xx`) 调整到了 `Y=28` 坐标处，使分数显示与页面标题横向垂直对齐，整体 UI 视觉更为整洁美观。

### 4.4 警笛式 LED 声光告警
* **正常运行状态**: LED2 处于 TIM2 CH1 渐变呼吸灯模式；AI 学习时 LED3 常亮，AI 监控时 LED4 常亮。
* **异常状态**: 若传感器掉线或 AI 判定温度异常突变，呼吸灯自动熄灭，板载告警灯 PC13 快速闪烁，LED3 与 LED4 会开启 5Hz 的红蓝警灯交替爆闪模式，进行强烈的声光报警。

### 4.5 高效 Flappy Bird 像素引擎
* **Dirty Rects 局部增量刷新**: 针对软件模拟 SPI 速度慢的问题，每帧（30ms）仅擦除和重绘像素变动区域（12×12 的黄色小鸟和 4 像素宽的管道变化边缘），让游戏能够在慢速总线下流畅稳定在 **30+ FPS**，毫无闪烁。
* **按键与触摸消抖缓存**: `game_key` 异步缓存机制将 10ms 消抖物理按键和触摸中断产生的 Tap 点击暂存，在 30ms 游戏主循环节中消费清零，消除了点击漏按的现象。

## 5. 快速编译与运行

1. **克隆项目**:
   ```bash
   git clone https://github.com/bestxby/micro-com-final.git
   ```
2. **启动 Keil**: 双击打开根目录下的 `demo.uvprojx`。
3. **确认编译器**: 
   * 打开 `Project` -> `Options for Target 'Target 1'` -> `Target` 选项卡；
   * 在 `ARM Compiler` 下选择 `Use default compiler version 5` 或 `Version 6`；
   * 确认 `C/C++` 选项卡中的 `C99 Mode` (对于 Version 5) 或 `Language C` -> `c99` (对于 Version 6) 已被勾选启用。
4. **编译项目**: 点击 **Build (F7)** 进行编译。
5. **下载烧录**: 连接好仿真器（如 DAP-Link 或 ST-Link），点击 **Download (F8)** 将程序烧录至 STM32 板中。
6. **触控与按键交互**:
   * 在主界面通过**左右划屏**或按开发板上的 **KEY1 (PB0)** 循环切换展示页面；
   * 在游戏页面，直接**轻触游戏画面中央**或按开发板上的 **KEY2 (PB8)** 控制小鸟飞行和游戏重启。
