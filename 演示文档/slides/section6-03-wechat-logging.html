物联网通信

云边端物联网通信与本地数据归档

ESP8266 WiFi, NTP Clock Sync, ServerChan WeChat Alerts, and SD Logging

01
WiFi 自动重连
与网络链路自愈
  • • 连接状态实时监控: 系统上电后通过 AT 指令自动连接热点，并在 LCD 屏幕上实时、动态刷新 ONLINE/OFFLINE 连接状态指示。
  • • 底层物理自愈: 当热点异常断开时，依靠 ESP8266 硬件底层重连算法持续轮询并重试，待热点重现后自动重连并重建通信链路。
02
网络时间与天气
实时自动获取
  • • 获取具体时空信息: 系统联网后自动通过 NTP 服务同步网络高精度北京时间（维护秒级 RTC），同时定时向天气服务端发起请求，直接获取当前城市的室外气温与当日的天气状况（多态晴雨图标/文本）。
  • • 极简原生解析: 保留底层单片机内存优化的精髓，强调使用标准的 strstr 对上述时间、天气网络数据流进行原生匹配，避免重型 JSON 解析库带来的开销与碎片。
03
512B 扇区对齐
SD 卡日志归档
  • • 底层对齐: 针对 FAT32 特性，设计了基于 512B 扇区对齐的日志写入算法。无论是读取还是追加覆写，均严格按照物理边界进行整块擦写，避免破坏底层簇链。
  • • 数据归档: 无论是状态机的每一次异常切换事件，还是带有精确北京时间戳的温湿度参数，都会被实时、永久地序列化保存到本地 SD 卡中。
04
微信警报推送
与防抖限制
  • • 触发条件: 一旦超声波雷达侦测到非法入侵，或本地环境温度发生剧烈突变 (>5℃)，将立即通过 Server酱 云端 API 向用户的绑定微信推送图文警报。
  • • 防抖机制: 为防止多次快速触发导致 HTTP 请求大量拥塞，系统在底层专门设计了一把状态机软锁（180s 物理冷却期），静默拦截一切短时间内的重复报警。
