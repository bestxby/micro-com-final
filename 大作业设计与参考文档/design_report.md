# 环境自适应智能守护与物联网交互系统综合设计报告 (System Design Report)

本报告详细阐述了基于 **STM32F103C8T6** 微控制器的大作业系统——“智能守护卫士与物联网大屏系统”的总体架构、硬件接口、底层驱动、自适应算法、图形界面渲染以及双固件展示技术方案。

---

## 1. 系统总体架构与设计思路

本系统基于微机原理与嵌入式系统设计方法，采用**分层、高内聚、低耦合**的模块化架构，分为硬件层、底层驱动与协议层、算法与数据处理层、系统控制与应用层。

```mermaid
graph TD
    subgraph 物理硬件层 (Physical Hardware)
        SENSORS[AHT20温湿度 / BMP280气压 / BH1750光强]
        HC_SR04[HC-SR04 超声波传感器]
        SD_CARD[SD 存储卡]
        ESP8266[ESP-01S WiFi 模块]
        DISP[ST7796S 4.0寸液晶屏 / TM1637数码管]
        BEEP_LED[有源蜂鸣器 / 三色指示灯]
        KEYS[BTN1 / BTN2 / KEY3 独立按键]
    end

    subgraph 底层驱动与总线层 (Bus Drivers)
        I2C_drv[软件模拟 I2C 总线]
        SPI_drv[软件模拟 SPI 总线]
        USART_drv[USART1 寄存器级驱动]
        TIM_drv[TIM2 三通道 PWM 驱动]
        GPIO_drv[寄存器级 GPIO 位带操作]
    end

    subgraph 算法与数据处理层 (Algorithm Layer)
        EMA_flt[EMA 指数滑动平均滤波器]
        AI_learn[无监督均值基准自适应学习]
        Healing[1.5s 传感器热插拔自愈重连]
        SectorLog[SD卡裸扇区日志追加与上电回溯]
    end

    subgraph 状态机与应用控制层 (Application Layer)
        UI_Mgr[6页科技感 UI 调度管理器]
        Game_Eng[Dirty Rects 增量刷新飞鸟游戏引擎]
        IoT_Mgr[NTP 北京时间解析 / 心知天气拉取]
        Alert_Mgr[防盗警戒状态机 / 微信 Server酱 推送]
    end

    SENSORS -->|I2C| I2C_drv
    SD_CARD -->|SPI| SPI_drv
    DISP -->|SPI / 二线制| SPI_drv
    ESP8266 -->|串口| USART_drv
    BEEP_LED -->|GPIO/PWM| TIM_drv
    KEYS -->|GPIO读| GPIO_drv

    I2C_drv & SPI_drv & USART_drv & TIM_drv & GPIO_drv --> EMA_flt
    EMA_flt --> AI_learn & SectorLog & Alert_Mgr
    AI_learn & SectorLog & Alert_Mgr --> UI_Mgr & Game_Eng & IoT_Mgr
```

## 3. 驱动层设计与片上资源优化

### 3.1 寄存器级位带加速 (Bit-banding) 与 JTAG 解锁
在 STM32F103C8T6 中，JTAG 默认锁定了 `PB3 (JTDO)` 和 `PB4 (JNTRST)`。为了释放这两个引脚用作 SD 卡的 SPI 数据线和片选线，系统在初始化阶段利用寄存器解锁技术配置了 AFIO 重映射，保留两线制 SWD 调试（PA13/PA14）的同时禁用了 JTAG 调试：
```c
RCC->APB2ENR |= RCC_APB2ENR_AFIOEN;
AFIO->MAPR = (AFIO->MAPR & ~AFIO_MAPR_SWJ_CFG) | AFIO_MAPR_SWJ_CFG_JTAGDISABLE;
```
同时，为了在软件模拟 SPI 中达到极高的读写效率，系统在 GPIO 控制中抛弃了标准库函数，直接采用 Cortex-M3 的 ODR/IDR **位带映射操作**。例如，引脚拉高拉低可在单周期指令内实现，将软件模拟 SPI 通信频率提升至约 2MHz 左右，为 LCD 局部刷新和 SD 卡高速写入提供了硬件支持。

### 3.2 寄存器级 USART1 中断环形接收缓冲区
为了保障 WiFi 异步通信数据不丢失，系统编写了基于控制寄存器的串口发送与接收中断服务函数，内置了一个 512 字节的接收环形缓冲区。
- 发送时直接写入 `USART1->DR` 并轮询等待 `TC` 位拉高。
- 接收时使用中断函数 `USART1_IRQHandler` 自动将数据放入缓存，在应用层循环取出，支持对大段网络天气和时间数据流的背景吞吐。

### 3.3 TIM2 三通道独立 PWM 指示灯驱动
配置定时器 TIM2 产生约 1kHz 的 PWM 脉波波形（`PSC=719`, `ARR=99`），并将通道 1 (PA0)、通道 2 (PA1)、通道 3 (PA2) 挂载对应的红、黄、绿状态灯。
- 在安全守护下，通过在 10ms 心跳节拍下平滑累加/递减 `TIM2->CCR3` 的值，令绿色指示灯呈现极其平滑的呼吸渐变效果。
- 异常时，状态机直接关闭 PWM 呼吸，切换到 CCRx 占空比的寄存器爆闪（写 0 或 99），直观传达系统层级的不同状态。

## 4. 算法层设计：AI 智能监测与自愈机制

### 4.1 指数移动平均 (EMA) 滤波器
传感器读取的原始温度 $T_{raw}$ 常因电路干扰产生毛刺。系统在算法层第一阶段部署了 EMA 指数滑动平均数字滤波器：
$$T_f[n] = \alpha \cdot T_{raw}[n] + (1 - \alpha) \cdot T_f[n-1]$$
其中平滑常数 $\alpha$ 被设置为 `0.2f`。该滤波算法无需维护庞大的历史采样序列，SRAM 占用为常数级 $O(1)$。它成功消除了高频毛刺，并对传感器数据进行了平滑，为 AI 基准判定提供了纯净的输入源。

### 4.2 无监督环境自适应基准学习算法
系统在上电后的前 100 次有效采样周期（约 150 秒）内进入“AI 学习状态”（`AI_STATE_LEARNING`）。系统不设定硬编码温度报警值，而是通过递推公式在线更新本底温度的动态基准值：
$$\mu_{base}[n] = \frac{n-1}{n}\mu_{base}[n-1] + \frac{1}{n}T_f[n]$$
采样满 100 次后锁定基准值 $\mu_{base}$。此后实时计算偏差 $\Delta T = |T_f - \mu_{base}|$：
- 若 $\Delta T \le 5^\circ\text{C}$：系统正常运行，绿色指示灯平滑呼吸。
- 若 $\Delta T > 5^\circ\text{C}$：系统判定当前环境温度突变，顶部状态栏高亮 `"ANOMALY ALARM"`，红色指示灯爆闪，并写入异常日志。此机制允许设备自适应空调房、室外等多种复杂工作环境。

### 4.3 传感器热插拔自愈自愈算法 (Self-Healing)
由于面包板和杜邦线在实验演示中易发生物理断联，系统为所有 I2C 传感器设计了“3次读取失败自动离线”的自愈逻辑。当读取失败 3 次时，系统自动将该传感器置为 `OFFLINE`；离线后，主循环在后台以 1.5s 的安全周期不断尝试调用 `Init` 重新激活传感器。一旦物理接线重连，系统可在 1.5s 内实现全自动热插拔自愈，防止系统因总线卡死。

### 4.4 底层裸扇区日志追加与自动回溯续写 (Raw Sector Logging)
脱离了 FAT 复杂文件系统，直接读写 SD 卡的底层物理扇区。系统规定每秒抓取传感器数据合成 64 字节的 ASCII 日志块，积累满 8 条（即 512 字节，对应一个物理扇区）后调用 `SD_WriteBlock` 一次性向 SD 卡追加写入。
- **自动回溯**：上电时，系统会自动从扇区 1000 开始往后扫描，读取扇区首字节并检查是否为 `'2'`（新北京时间日志首字）或 `'U'`（旧系统运行时间日志首字）。若发现该扇区为空白，则判定其为本次开机日志写入的起始位置。此设计有效防止了断电重启后覆盖历史日志的问题。

## 5. UI 与游戏引擎设计

### 5.1 现代 Cyberpunk 暗黑霓虹 / Claude 暖白双主题调度
系统内建两套风格特异的主题方案：
- **Cyberpunk HUD 暗黑霓虹主题**：以深黑为背景（`0x0000`），辅以 Slate 灰（`0x2104`）卡片和高亮霓虹蓝、绿色文本，在夜间环境下展现极强的科技质感。
- **Claude Paper 温润明亮主题**：背景采用温暖奶油纸张色（`0xF7BD`），配以纯白卡片和深褐文本，在白天日光下提供极佳的文字易读性。
按 BTN2 (PB8) 键可瞬时反转全局色彩配置，并触发当前页面的局部区域重画。

### 5.2 脏矩形 (Dirty Rects) 局部增量游戏刷新技术
在 Page 3 飞鸟游戏中，如果采用传统全屏刷屏方式，一帧 480×320 像素的 RGB565 数据为 307.2 KB。在 72MHz 下模拟 SPI 的最大有效带宽为 1.25 Mbps，传输全屏像素耗时高达 2.0 秒，导致游戏帧率无法突破 0.5 FPS，且闪烁剧烈。
为了解决这一物理总线限制，系统设计了**脏矩形（Dirty Rects）局部增量刷新机制**：
- 小鸟（12×12 像素）上下运动时，仅使用背景色填充旧坐标的 12×12 像素区域（擦除），并重绘新坐标的 12×12 像素黄色鸟。
- 管道（32 像素宽，230 像素高）向左滚动时，并不重绘整个管道，而仅仅用背景色擦除右侧由于平移空出来的 4px 像素，并用绿色在新移入的最左侧填充 4px 像素条。
这使游戏单帧的像素写入量由 307.2 KB 骤降至仅约 **3.9 KB**，数据量减少了 **98.7%**。这使系统在 30ms 的定时刷新节拍下能够稳定提供 **30+ FPS** 的无闪烁流畅动作画面。

## 6. Git 双分支模块化展示设计 (Dual-Build Architecture)

为了在紧张的引脚限制下同时展示 SD 卡绝对时钟日志以及液晶屏触控手势功能，我们利用 **Git 分支管理** 提供两个独立的固件版本：

```
                              [智能守护系统代码库]
                                       |
                   +-------------------+-------------------+
                   |                                       |
         [master 分支：物联网报警]             [feat-touch 分支：屏幕触控交互]
                   |                                       |
     - 启用 SD卡 (CS: PB4, MISO: PA11)      - 禁用 SD卡 避免 SPI 电气短路
     - 禁用液晶屏触摸芯片 XPT2046          - 启用触摸屏驱动 (PB3/4/5/PA11/PA12)
     - 支持 NTP 北京时间 SD 日志存储        - 支持屏幕底部 [PREV]/[NEXT] 触点翻页
     - 支持超声波测距与三级布防             - 支持 [RST] 点击重置 AI 学习基准
     - 支持微信 Server酱 实时云端推送      - 支持 [+][-] 触点修改防盗距离阈值
     - 支持数码管 TM1637 本地温度显示      - 支持点击屏幕控制 Flappy Bird 振翅起飞
```

该“双分支、免重连、双烧录”的方案通过纯软件的分支配置切换，在不需要改动面包板任何物理接线的前提下，成功向验收老师清晰展示了所有大作业功能要求。

## 7. 系统级联合测试与验证分析

### 7.1 ESP8266 物联网自动校时与气象同步测试
上电后，串口与 ESP8266 进行 AT 指令通信，并成功连接到名称为“小小小波”的 2.4 GHz 个人热点。
- 联网成功后，TFT LCD 顶部指示灯状态更新，Page 5 的 `SSID` 变为 `小小小波`。
- 向 `cn.ntp.org.cn` 同步网络时间成功，系统自动将英文月份（如 Jun）格式化为数字北京时间 `2026-06-05 20:30:15` 并开始按秒高精度跳动，证明软件 RTC 本地递增对齐正常。
- 周期性拉取心知天气 API 成功，Page 5 右侧卡片准确更新室外气温 `Temp: 28.00 C`，室外天气显示 `Sunny`，位置同步显示 `Guangzhou`。

### 7.2 警戒布防声光入侵报警与微信 Server酱 推送测试
1. 在任意页面按下面包板按键 KEY3 (PB9)，系统进入 ARMED 布防模式。
2. 当障碍物距离 HC-SR04 超声波探头大于 50cm 时，绿色 LED 亮起（安全）。
3. 移近手掌至 35cm，绿色 LED 熄灭，黄色 LED 亮起（黄色预警）。
4. 迅速移近手掌至 10cm（入侵距离 < 15cm）：
   - 红、黄、绿三盏指示灯同时以 5Hz 频率进行剧烈同步爆闪。
   - 5V 有源蜂鸣器 (PB15) 发出大分贝的刺耳连续警报声。
   - 微信上同步收到来自 Server酱 发送的推送消息：`【警报】防盗入侵触发！当前距离: 10.0cm`。
5. 再次按下 KEY3 撤防，蜂鸣器瞬时切断，微信防重入 3 分钟冷却机制生效（警报触发不轰炸微信，但本地声光与屏幕警告仍旧正常响应）。

### 7.3 SD 日志绝对时间戳数据校验
使用 Python 脚本 `Scripts/read_sd.py` 通过串口读取 SD 读卡器内的物理扇区 1000 开始的数据。提取的部分日志片段如下：
```text
2026-06-05 20:30:40 T: 25.40C H: 50.20% L: 120.5lx A:0 M:0
2026-06-05 20:30:48 T: 25.40C H: 50.18% L: 120.3lx A:0 M:0
2026-06-05 20:30:56 T: 30.52C H: 55.40% L: 119.8lx A:2 M:1
```
其中，最后一行的 `A:2` 表明此时 AI 处于异常报警状态（`AI_STATE_ANOMALY`），`M:1` 表明系统当前处于警戒布防状态。绝对时间戳和环境数据准确地以 64 字节大小写入，数据读写稳定性优良。

## 8. 心得体会与实验总结

通过本次《微机原理与嵌入式系统实验》期末大作业的设计，我不仅掌握了完整的嵌入式系统全栈开发流程，更对以下几点有了更深的工程体会：
1. **算法对物理性能的提升作用**：在软件模拟 SPI 总线的速度受限情况下，通过设计“脏矩形（Dirty Rects）”算法实现了 30+ FPS 流畅游戏渲染，说明良好的算法能有效弥补硬件的通信速率瓶颈。
2. **边缘智能的自适应价值**：设计的无监督基准学习算法摒弃了传统固定阈值的不足，能够让监测器自行适配不同工作场所的环境温度，体现了边缘端设备的高适应性。
3. **分合有道的 Git 开发模式**：利用 Git 双分支管理，成功在单套物理连线下面板化演示了两套完全重合的硬件功能模块（SD 卡绝对日志存储 vs 触控人机调参小游戏），提高了资源极度紧缺下的原型测试效率。

本报告中所有功能规格及运行现象均通过实物联调与验收验证，系统工作稳定，达到了预期的设计指标。
