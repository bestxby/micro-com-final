# 工程文档 Version1 — 目录结构与 STM32 引脚资源映射

> **项目名称**: 基于 STM32F103C8T6 的传感器监测系统 (AI 自适应异常检测)  
> **目标芯片**: STM32F103C8T6 (Cortex-M3, 64KB Flash, 20KB SRAM)  
> **开发环境**: Keil uVision 5 + ARM Compiler 6 (AC6)  
> **实验平台**: EES-351-MC 微机原理实验箱 (EES-351F-V1.0 STM32 子板)  
> **文档版本**: Version1  
> **更新日期**: 2026-05-29

## 2. STM32F103C8T6 引脚资源分配总览

以下按功能模块详细列出每个 GPIO 引脚的用途、连接目标及硬件配置。

### 2.1 LED 指示灯模块

| LED 编号 | STM32 引脚 | 连接目标 | 有效电平 | GPIO 模式 | 功能描述 |
|:---:|:---:|:---|:---:|:---|:---|
| **LED1** | **PC13** | 核心板载 LED | **低电平 (0)** | 推挽输出, 50MHz | 状态指示灯：系统正常与异常时均保持关闭（常灭，保留备用） |
| **LED2** | **PA0** | 面包板外接 LED (红) | **高电平 (1)** | 复用推挽输出, 50MHz | **红色报警/故障指示灯**：撤防下硬件故障常亮，温度异常（突变偏差超过±5℃）时以 5Hz 频率闪烁；布防下入侵报警（距离<15cm）时以 5Hz 频率闪烁 |
| **LED3** | **PA1** | 面包板外接 LED (黄) | **高电平 (1)** | 复用推挽输出, 50MHz | **黄色学习/预警指示灯**：撤防下 AI 正在进行 150 秒的环境基准温度学习时常亮；布防下，超声波异常时 5Hz 闪烁，距离 15~50cm 预警常亮，入侵时 5Hz 闪烁 |
| **LED4** | **PA2** | 面包板外接 LED (绿) | **高电平 (1)** | 复用推挽输出, 50MHz | **绿色守护/运行指示灯**：撤防下 AI 学习结束、系统安全监控时呈现平滑呼吸渐变效果；布防下，距离 > 50cm 安全常亮，入侵时 5Hz 闪烁 |

> **设计说明**: LED2/3/4 (PA0/1/2) 引脚配置为 TIM2 通道 1/2/3 的复用推挽输出模式。TIM2 配置参数：PSC=719, ARR=99，产生约 1kHz 的 PWM 载波频率。绿灯的呼吸效果通过 `LED_ProcessBreathing()` 在主循环中逐步递增/递减 CCR3 占空比寄存器实现。

### 2.2 物理按键与布防模块

| 按键编号 | STM32 引脚 | 连接目标 | 触发方式 | GPIO 模式 | 功能描述 |
|:---:|:---:|:---|:---:|:---|:---|
| **KEY1** | **PB0** | 开发板 BTN1 按键 | **低电平按下** | 上拉输入 | **UI 页面切换**：在 Page 0 (实时监测) → Page 1 (AI自适应) → Page 2 (历史日志) → Page 3 (飞鸟游戏) → Page 4 (防盗安全) → Page 5 (网络天气) 之间循环切换 |
| **KEY2** | **PB8** | 开发板 BTN2 按键 | **低电平按下** | 上拉输入 | **多功能键**：非游戏页面下为“主题切换”和“参数/自愈重置”复合键（重置 AI 基准、清零计数器、重新初始化 AHT20）；游戏页面下为小鸟飞跃跳跃键 |
| **KEY3** | **PB9** | 面包板外接按键 | **低电平按下** | 上拉输入 | **一键布防/撤防**：用于一键开启/关闭防盗监测模式 |

> **消抖与复用机制**: 采用软件延时消抖（20ms 确认 + 释放锁定标志位）。KEY2 在游戏运行时（Page 3）不触发页面重置或主题切换，仅响应小鸟跳跃动作，规避了状态竞争。

### 2.3 LCD 彩色显示屏模块

> **显示器件**: MSP4020 4.0 inch SPI TFT LCD  
> **驱动 IC**: ST7796S (兼容 ILI9341 指令集)  
> **分辨率**: 320 × 480 (横屏模式, RGB565 色深)  
> **接口**: 4 线软件模拟 SPI (SCK + SDA/MOSI + CS + DC/RS) + RST + BL

| 信号名称 | STM32 引脚 | 连接目标 | GPIO 模式 | 功能描述 |
|:---|:---:|:---|:---|:---|
| **LCD_RST** | **PA3** | 板载 A3 排针 → TFT RST | 推挽输出, 50MHz | 液晶屏硬件复位：上电时产生 100ms 低电平复位脉冲 |
| **LCD_CS** | **PA4** | 板载 A4 排针 → TFT CS | 推挽输出, 50MHz | SPI 片选信号：低电平选中 LCD 控制器，每次 SPI 事务开始时拉低、结束时拉高 |
| **LCD_SCL** | **PA5** | 板载 A5 排针 → TFT SCK | 推挽输出, 50MHz | SPI 时钟线：软件模拟串行时钟，上升沿锁存数据 |
| **LCD_DC** | **PA8** | 板载 B3 排针 → TFT DC | 推挽输出, 50MHz | 命令/数据选择 (RS)：低电平=命令寄存器，高电平=显示数据 |
| **LCD_SDA** | **PA7** | 板载 B4 排针 → TFT SDA | 推挽输出, 50MHz | SPI MOSI 数据线：软件逐位串行输出 8-bit 命令/数据字节 |
| **LCD_BL** | **PB1** | 板载 D6 排针 → TFT LED | 推挽输出, 50MHz | 背光控制：高电平点亮背光，初始化完成后置高常亮 |

> **SPI 时序**: 采用软件位操作 (bit-banging) 方式模拟 SPI 协议，数据在 SCK 上升沿被从机采样（CPOL=0, CPHA=0 模式等效）。写命令时 DC=0，写数据时 DC=1。

### 2.4 I²C 总线模块 (软件模拟)

> **总线类型**: 软件模拟 I²C (bit-banging)，非 STM32 硬件 I²C 外设  
> **通信速率**: 约 100~200 kHz (软件延时调谐)  
> **挂载设备**: AHT20 (地址 0x38) + BMP280 (地址 0x76) + BH1750 (地址 0x23)

| 信号名称 | STM32 引脚 | 连接目标 | GPIO 模式 | 功能描述 |
|:---|:---:|:---|:---|:---|
| **I2C_SCL** | **PA6** | 非 Arduino 排针 → 传感器 SCL | 开漏输出, 50MHz | I²C 时钟线：主机控制，产生起始/停止条件及时钟脉冲 |
| **I2C_SDA** | **PB10** | 非 Arduino 排针 → 传感器 SDA | 开漏输出, 50MHz | I²C 数据线：双向数据传输，收发切换时通过 GPIO 模式 + IDR 读取实现 |

> **设计说明**: SCL 和 SDA 均配置为开漏输出模式，空闲状态拉高。数据收发方向切换通过 `I2C_SDA_H()` 释放总线 + `I2C_SDA_READ()` 读取 IDR 寄存器实现。两条总线共享同一组 I²C 引脚，通过不同的 7 位从机地址区分 AHT20 (0x38) 和 BMP280 (0x76)。

---

### 2.5 传感器子模块

#### 2.5.1 AHT20 温湿度传感器

| 参数 | 规格 |
|:---|:---|
| **挂载总线** | I²C (软件模拟, PA6/PB10) |
| **7 位从机地址** | `0x38` (写地址 0x70, 读地址 0x71) |
| **测量范围** | 温度 -40 ~ +85°C, 湿度 0 ~ 100%RH |
| **典型精度** | 温度 ±0.3°C, 湿度 ±2%RH |
| **数据输出** | 20 位原始 ADC (湿度) + 20 位原始 ADC (温度), 通过公式转换为浮点物理量 |
| **初始化流程** | I²C 初始化 → 延时 40ms → 读取状态字节 → 若 bit3=0 发送 0xBE/0x08/0x00 校准命令 |
| **测量触发** | 发送 0xAC/0x33/0x00 → 等待 80ms → 轮询状态 bit7 (忙标志) → 读取 6 字节数据 |

#### 2.5.2 BMP280 气压传感器

| 参数 | 规格 |
|:---|:---|
| **挂载总线** | I²C (软件模拟, PA6/PB10) |
| **7 位从机地址** | `0x76` (写地址 0xEC, 读地址 0xED, SDO 引脚接地) |
| **测量范围** | 气压 300 ~ 1100 hPa, 温度 -40 ~ +85°C |
| **典型精度** | 气压 ±0.12 hPa, 温度 ±0.5°C |
| **数据输出** | 20 位原始 ADC (气压) + 20 位原始 ADC (温度), 经过出厂校准系数补偿后输出浮点物理量 |
| **初始化流程** | 读取 Chip ID (0xD0 → 应为 0x58) → 软复位 (0xE0 ← 0xB6) → 读取 24 字节校准参数 (0x88~0x9F) → 配置 Ctrl (0xF4 ← 0x37: Normal 模式, 温度×1, 气压×16) → 配置 Config (0xF5 ← 0x48: 125ms 待机, IIR×4) |

## 7. 系统功能概述

本项目实现了一个**基于边缘端 AI 自适应学习及 IoT 联网的环境监测与防盗警戒系统**，核心功能包括：

1. **多外设综合感知与显示**: 支持 AHT20 温湿度、BH1750 光强、BMP280 气压、HC-SR04 超声波多路传感器感知，以及 4.0 inch LCD 屏、TM1637 数码管双看板显示。
2. **传感器热插拔自愈重连**: 对所有传感器设计了 3 次容错自愈算法，掉线后自动在后台尝试重初始化恢复。
3. **AI 自适应基准学习**: 引入 EMA 平滑滤波并自适应学习环境基准温度，在温度偏离基线超过 ±5°C 时触发突变异常报警。
4. **IoT 联网时钟与天气**: 利用 ESP8266 (ESP-01S) 连接个人热点，获取 NTP 时钟同步并动态解析本地（广州）室外天气实况。
5. **WeChat 微信警报推送**: 发生安全入侵（距离 < 15cm）或温度突变异常时，利用 Server酱 进行云端推送，具备 180 秒冷却防风暴机制。
6. **六页 UI 科技交互**: Page 0 实时参数, Page 1 AI自适应曲线, Page 2 历史日志查询, Page 3 飞鸟小游戏, Page 4 防盗安全卫士（带测距趋势图）, Page 5 网络时钟与天气。
7. **三色指示灯与有源警报**: 
   - 撤防/正常模式：绿色 LED (PA2) 呈现平滑呼吸渐变效果，其余常灭。
   - 撤防/异常模式：红色 LED (PA0) 5Hz 频率闪烁，其余常灭，蜂鸣器静音。
   - 布防模式：绿色/黄色/红色指示灯指示 3 级距离警戒；入侵时三灯同步 5Hz 爆闪且蜂鸣器 (PB15) 长鸣警告。
8. **裸扇区数据追加与自动回溯**: 脱离文件系统，以 64 字节块向 SD 卡扇区 1000 开始追加绝对时间数据日志，支持上电自动回溯续写。
