# 工程文档 Version1 — 目录结构与 STM32 引脚资源映射

> **项目名称**: 基于 STM32F103C8T6 的传感器监测系统 (AI 自适应异常检测)  
> **目标芯片**: STM32F103C8T6 (Cortex-M3, 64KB Flash, 20KB SRAM)  
> **开发环境**: Keil uVision 5 + ARM Compiler 6 (AC6)  
> **实验平台**: EES-351-MC 微机原理实验箱 (EES-351F-V1.0 STM32 子板)  
> **文档版本**: Version1  
> **更新日期**: 2026-05-29

## 2. STM32F103C8T6 引脚资源分配总览

以下按功能模块详细列出每个 GPIO 引脚的用途、连接目标及硬件配置。

### 2.1 LED 指示灯模块

| LED 编号 | STM32 引脚 | 连接目标 | 有效电平 | GPIO 模式 | 功能描述 |
|:---:|:---:|:---|:---:|:---|:---|
| **LED1** | **PC13** | 核心板载 LED / 底板 LEDx | **低电平 (0)** | 推挽输出, 50MHz | 异常告警指示灯：系统正常时常灭；检测到传感器异常时以约 5Hz 频率快速闪烁 |
| **LED2** | **PA0** | 板载 A0 排针 → LEDx | **高电平 (1)** | 复用推挽输出, 50MHz | **TIM2_CH1 PWM 呼吸灯**：系统正常时呈现平滑呼吸渐变效果（~720Hz PWM）；异常时强制熄灭 |
| **LED3** | **PA1** | 板载 A1 排针 → LEDx | **高电平 (1)** | 推挽输出, 50MHz | 预留外接 LED (当前未参与逻辑控制) |
| **LED4** | **PA2** | 板载 A2 排针 → LEDx | **高电平 (1)** | 推挽输出, 50MHz | 预留外接 LED (当前未参与逻辑控制) |

> **设计说明**: LED2 的 PA0 引脚配置为 TIM2 通道 1 的复用推挽输出模式。TIM2 配置参数：PSC=999, ARR=99，产生约 720Hz 的 PWM 载波频率。呼吸效果通过 `LED_ProcessBreathing()` 在主循环中逐步递增/递减 CCR1 占空比寄存器实现。

---

### 2.2 独立按键输入模块

| 按键编号 | STM32 引脚 | 连接目标 | 触发方式 | GPIO 模式 | 功能描述 |
|:---:|:---:|:---|:---:|:---|:---|
| **KEY1** | **PB0** | 板载 D0 排针 → 底板 BTN1 | **低电平按下** | 上拉输入 | **UI 页面切换**：在实时监测 (Page 0) → AI 学习状态 (Page 1) → 历史日志 (Page 2) 之间循环切换 |
| **KEY2** | **PB8** | 板载 D7 排针 → 底板 BTN2 | **低电平按下** | 上拉输入 | **系统复位**：重置 AI 检测器学习参数、清零失败计数器、强制重新初始化 AHT20/BMP280 传感器 |

> **消抖策略**: 采用软件延时消抖（20ms 确认 + 释放锁定标志位），默认工作在单次触发模式（`mode=0`），防止长按重复触发。

---

### 2.3 LCD 彩色显示屏模块

> **显示器件**: MSP4020 4.0 inch SPI TFT LCD  
> **驱动 IC**: ST7796S (兼容 ILI9341 指令集)  
> **分辨率**: 240 × 320 (竖屏模式, RGB565 色深)  
> **接口**: 4 线软件模拟 SPI (SCK + SDA/MOSI + CS + DC/RS) + RST + BL

| 信号名称 | STM32 引脚 | 连接目标 | GPIO 模式 | 功能描述 |
|:---|:---:|:---|:---|:---|
| **LCD_RST** | **PA3** | 板载 A3 排针 → TFT RST | 推挽输出, 50MHz | 液晶屏硬件复位：上电时产生 100ms 低电平复位脉冲 |
| **LCD_CS** | **PA4** | 板载 A4 排针 → TFT CS | 推挽输出, 50MHz | SPI 片选信号：低电平选中 LCD 控制器，每次 SPI 事务开始时拉低、结束时拉高 |
| **LCD_SCL** | **PA5** | 板载 A5 排针 → TFT SCK | 推挽输出, 50MHz | SPI 时钟线：软件模拟串行时钟，上升沿锁存数据 |
| **LCD_DC** | **PA6** | PMOD 排针 → TFT DC | 推挽输出, 50MHz | 命令/数据选择 (RS)：低电平=命令寄存器，高电平=显示数据 |
| **LCD_SDA** | **PA7** | 板载 B1/SDA 排针 → TFT SDA | 推挽输出, 50MHz | SPI MOSI 数据线：软件逐位串行输出 8-bit 命令/数据字节 |
| **LCD_BL** | **PB1** | 板载 D1 排针 → TFT LED | 推挽输出, 50MHz | 背光控制：高电平点亮背光，初始化完成后置高常亮 |

> **SPI 时序**: 采用软件位操作 (bit-banging) 方式模拟 SPI 协议，数据在 SCK 上升沿被从机采样（CPOL=0, CPHA=0 模式等效）。写命令时 DC=0，写数据时 DC=1。

---

### 2.4 I²C 总线模块 (软件模拟)

> **总线类型**: 软件模拟 I²C (bit-banging)，非 STM32 硬件 I²C 外设  
> **通信速率**: 约 100~200 kHz (软件延时调谐)  
> **挂载设备**: AHT20 (地址 0x38) + BMP280 (地址 0x76)

| 信号名称 | STM32 引脚 | 连接目标 | GPIO 模式 | 功能描述 |
|:---|:---:|:---|:---|:---|
| **I2C_SCL** | **PB6** | 板载 D5 排针 → AHT20 SCL / BMP280 SCK | 开漏输出, 50MHz | I²C 时钟线：主机控制，产生起始/停止条件及时钟脉冲 |
| **I2C_SDA** | **PB7** | 板载 D6 排针 → AHT20 SDA / BMP280 SDI | 开漏输出, 50MHz | I²C 数据线：双向数据传输，收发切换时通过 GPIO 模式 + IDR 读取实现 |

> **设计说明**: SCL 和 SDA 均配置为开漏输出模式，空闲状态拉高。数据收发方向切换通过 `I2C_SDA_H()` 释放总线 + `I2C_SDA_READ()` 读取 IDR 寄存器实现。两条总线共享同一组 I²C 引脚，通过不同的 7 位从机地址区分 AHT20 (0x38) 和 BMP280 (0x76)。

---

### 2.5 传感器子模块

#### 2.5.1 AHT20 温湿度传感器

| 参数 | 规格 |
|:---|:---|
| **挂载总线** | I²C (软件模拟, PB6/PB7) |
| **7 位从机地址** | `0x38` (写地址 0x70, 读地址 0x71) |
| **测量范围** | 温度 -40 ~ +85°C, 湿度 0 ~ 100%RH |
| **典型精度** | 温度 ±0.3°C, 湿度 ±2%RH |
| **数据输出** | 20 位原始 ADC (湿度) + 20 位原始 ADC (温度), 通过公式转换为浮点物理量 |
| **初始化流程** | I²C 初始化 → 延时 40ms → 读取状态字节 → 若 bit3=0 发送 0xBE/0x08/0x00 校准命令 |
| **测量触发** | 发送 0xAC/0x33/0x00 → 等待 80ms → 轮询状态 bit7 (忙标志) → 读取 6 字节数据 |

#### 2.5.2 BMP280 气压传感器

| 参数 | 规格 |
|:---|:---|
| **挂载总线** | I²C (软件模拟, PB6/PB7) |
| **7 位从机地址** | `0x76` (写地址 0xEC, 读地址 0xED, SDO 引脚接地) |
| **测量范围** | 气压 300 ~ 1100 hPa, 温度 -40 ~ +85°C |
| **典型精度** | 气压 ±0.12 hPa, 温度 ±0.5°C |
| **数据输出** | 20 位原始 ADC (气压) + 20 位原始 ADC (温度), 经过出厂校准系数补偿后输出浮点物理量 |
| **初始化流程** | 读取 Chip ID (0xD0 → 应为 0x58) → 软复位 (0xE0 ← 0xB6) → 读取 24 字节校准参数 (0x88~0x9F) → 配置 Ctrl (0xF4 ← 0x37: Normal 模式, 温度×1, 气压×16) → 配置 Config (0xF5 ← 0x48: 125ms 待机, IIR×4) |

## 7. 系统功能概述

本项目实现了一个**基于 AI 自适应基准学习的温度异常检测系统**，核心功能包括：

1. **双传感器数据采集**: 通过软件 I²C 总线周期性轮询 AHT20 (温湿度) 和 BMP280 (气压/温度)，采样间隔约 1.5 秒。
2. **传感器热插拔自愈**: 连续 3 次通信失败自动标记掉线；掉线后每次轮询周期自动尝试重新初始化传感器以恢复连接。
3. **AI 自适应基准学习**: EMA (指数移动平均) 数字滤波 + 渐进式环境温度基准学习（100 个样本约 150 秒），学习完成后自动进入异常监测模式。
4. **实时异常检测**: 滤波后温度偏离学习基准超过 ±5°C 时触发异常告警。
5. **三页 UI 交互**: Page 0 实时监测数据面板, Page 1 AI 学习状态 + 偏差折线图 + 进度条, Page 2 历史异常日志记录 (环形缓冲区, 最多 5 条)。
6. **双 LED 状态指示**: 正常模式 — PA0 呼吸灯渐变；异常模式 — PC13 高频闪烁 (5Hz)。
