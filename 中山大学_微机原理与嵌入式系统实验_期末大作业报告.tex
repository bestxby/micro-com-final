\section{摘要与项目背景}

\subsection{摘要}
本项目设计并实现了一套基于 \texttt{STM32F103C8T6} 微控制器的物联网自适应环境监测、声光警戒与多维人机交互系统。
系统在硬件架构上高度整合了多种传感与外设模块：以 AHT20、BMP280 及 BH1750 为多参数环境感知核心，通过开漏输出的软件模拟 I2C 总线实现传感器数据采集；部署了 ESP8266 (ESP-01S) 无线模块接入互联网，以 USART1 中断环形接收缓存实现数据吞吐；集成了 HC-SR04 超声波测距传感器、TM1637 四位数码管、5V 有源分体蜂鸣器及 SPI SD 存储卡，构建了完整的声光防盗报警与绝对时间戳日志记录平台。

在算法层，系统引入了轻量级片上无监督统计学习机制，在上电前 150 秒自适应迭代更新当前环境的稳定温度基准线 $\mu_{base}$，并在检测周期部署指数滑动平均滤波器（EMA，$\alpha=0.2$）平滑波动；当滤波温度偏离基线超过 $\pm 5^\circ\text{C}$ 时，触发异常警灯交替爆闪。在防盗警戒状态下，超声波测距提供三级安全指示，入侵（距离 $<15\text{cm}$）触发蜂鸣器啸叫及微信（Server酱）云端推送，并配合 180 秒冷却机制防止消息轰炸。系统在 SD 卡扇区 1000 开始以 64 字节块追加绝对北京时间戳数据日志，具备上电扇区回溯续写与 I2C 链路 1.5 秒热插拔故障自愈机制。

为提升交互品质，系统设计了暗黑霓虹与亮色纸张两套触控/按键可切主题，提供示波器风格网格折线图。此外，系统内置了基于重力物理模型的 ``Flappy Bird'' 动作游戏。为解决软件模拟 SPI 的带宽瓶颈，提出了“脏矩形（Dirty Rects）”局部增量刷新技术，将每帧渲染数据量骤降 98.7\%，在 72MHz 主频下实现了 30+ FPS 的无闪烁极速运行。为了克服芯片引脚紧张所导致的 SPI 物理冲突（SD 卡与触摸屏共用管脚），系统采用 Git 双分支管理架构，提供物联网报警综合固件（\texttt{master}）和触控交互游戏固件（\texttt{feat-touch}），在无需改动物理连线的前提下，实现模块化演示，极大提高了嵌入式开发效率。

\subsection{项目背景与现实意义}
随着物联网与智能边缘计算的普及，环境监测终端已由传统的单一数据收集演变为集多维感知、云端警报、人性化交互于一体的智能网关。然而，静态硬编码阈值的报警方法极易受不同安装位置（如近散热风扇、阳光直射区）的基准差异干扰，产生频繁误报。此外，传统终端在常规监控时主控算力与显示屏常处于静止闲置状态，造成资源冗余。

本项目的研究旨在通过无监督的片上统计学习使终端具备对微气候的自学习能力，并整合 WiFi 网络天气获取及微信远程推送等 IoT 业务；同时通过脏矩形算法在低带宽下实现高性能游戏交互，为嵌入式系统的多协议共存、硬件瓶颈算法弥补以及低能耗交互界面设计提供了一种具有高度参考价值的解决方案。

\subsection{关键引脚重映射与 JTAG 解锁}
由于系统包含 LCD、SD卡、WiFi、超声波、数码管、按键等十余种外设，引脚资源极其紧张。其中，\texttt{PB3 (JTDO)} 和 \texttt{PB4 (JNTRST)} 上电后默认作为 JTAG 接口工作，这会锁死引脚电平导致 SD卡 的 SPI 通信失效。

为了释放它们作为普通 GPIO 使用，程序在初始化阶段对 AFIO 重映射寄存器进行了寄存器级配置，在安全保留 SWD 仿真下载端口（PA13/PA14）的同时禁用了 JTAG 调试，其寄存器配置代码如下：
\begin{lstlisting}[style=cstyle]
// 开启复用时钟并配置 SWJ 寄存器：保留两线 SWD 调试，解锁 PB3 和 PB4 供普通 GPIO 使用
RCC->APB2ENR |= RCC_APB2ENR_AFIOEN;
AFIO->MAPR = (AFIO->MAPR & ~AFIO_MAPR_SWJ_CFG) | AFIO_MAPR_SWJ_CFG_JTAGDISABLE;
\end{lstlisting}
配置完成后，PB3 与 PB4 分别成功用作 SD 卡的 MOSI 数据线与 CS 片选线，保证了 SPI 物理总线的通信。

\section{软件模块化设计与外设驱动层实现}

\subsection{软件模拟 I2C 总线总线控制}
由于 AHT20、BMP280 和 BH1750 三款传感器挂载于同一总线上，系统编写了基于控制寄存器的开漏模拟 I2C 驱动。通过配置寄存器 \texttt{GPIOA->CRL} 与 \texttt{GPIOB->CRH}，将 PA6 (SCL) 和 PB10 (SDA) 配置为 50MHz 模拟开漏输出模式，端口配置寄存器如下：
\begin{lstlisting}[style=cstyle]
// 清空控制位，并配置 PA6(SCL) 为模拟开漏，PB10(SDA) 为模拟开漏
GPIOA->CRL = (GPIOA->CRL & ~0x0F000000) | 0x07000000;
GPIOB->CRH = (GPIOB->CRH & ~0x00000F00) | 0x00000700;
\end{lstlisting}
在时钟控制上，利用 \texttt{Delay} 函数产生约 $150\text{kHz}$ 的总线频率，保证了通信时序建立时间与保持时间的裕量，有效杜绝了因寄生电容导致的数据读写竞争冒险。

\subsection{ESP8266 WiFi 驱动与 IoT 联网机制}
系统底层配置串口 USART1（波特率 115200bps, 8-N-1）并开启中断接收。在应用层编写了发送 AT 命令状态机。
1. **热点自动握手**：上电发送 \texttt{AT+CWMODE\_DEF=1} 配置为客户端模式，再通过 \texttt{AT+CWJAP\_DEF} 传入账户密码联接指定的 2.4 GHz 个人热点。
2. **NTP 北京时间获取**：向 NTP 服务器（\texttt{cn.ntp.org.cn}）发送请求，接收返回的英文时间数据（如 \texttt{Fri Jun 05 20:45:12 2026}），直接在接收缓冲区使用 \texttt{sscanf} 定位并格式化解析出标准的北京时间并更新本地时间寄存器。
3. **心知天气 API 拉取**：每 10 分钟自动建立 TCP 链接请求心知天气 API 接口（\texttt{api.seniverse.com}），接收 JSON 文本后直接通过 `strstr` 匹配 `"text":"` 和 `"temperature":"` 关键字字段，提取出室外温度与天气状况（如 Sunny、Rainy），避免了复杂的 JSON 解析库，大幅节约了片上 SRAM 开销。

\subsection{SD 卡绝对时间戳日志记录与扇区扫描算法}
SD卡底层基于软件模拟 SPI 总线进行块读写（扇区 1000 开始）。为了能够实现持久的断电保护，系统设计了 64 字节的 ASCII 数据块，并将同步自网络的北京绝对时间戳注入日志中：
\begin{lstlisting}[style=cstyle]
// 64 字节 ASCII 绝对时间数据块样例：
"2026-06-05 20:45:12 T:25.40C H:50.00% L:120.0lx A:0 M:0\r\n"
\end{lstlisting}
- **扇区扫描**：上电时系统轮询 SD卡，以 `SD\_ReadBlock` 从扇区 1000 逐扇区向后读取前 3 字节，若读取出的内容为字符 `'2'`（新时间戳）或 `'U'`（旧日志首字符），则表示该扇区已有记录，指针自动加 1。直到读到首字为空白物理块，锁定此坐标作为写入点，支持数据上电自动回溯续写，只增不减。
- **缓存双写**：主循环每一秒将当前数据组装至 `log\_buffer` 的 64 字节切片中。当积累满 8 条数据（满 512 字节，对应一个物理扇区）时，一次性写入 SD 卡，极大降低了 SPI 模拟总线高频读写对 CPU 的阻塞占用。

\section{算法层设计：AI 智能监测与自愈重连}

\subsection{指数滑动平均 (EMA) 数字滤波器}
为清除高频电磁毛刺与采样热噪声，采集得到的原始温度 $T_{raw}$ 自动被送入 EMA 数字滤波器中：
$$T_f[n] = \alpha \cdot T_{raw}[n] + (1 - \alpha) \cdot T_f[n-1]$$
其中平滑因子 $\alpha$ 被设置为 `0.2f`。该算法由于不需要在片上维护多维数组，存储空间复杂度为 $O(1)$，对瞬时干扰具备优异的 $-14\text{ dB}$ 衰减特性，同时滤波滞后低于 1.5s，能够提供高平滑且低延迟的滤波数据集。

\subsection{环境温度无监督基准自适应学习}
当系统初始化或按下 KEY2 重置时，AI 检测器处于学习状态（\texttt{AI\_STATE\_LEARNING}）。在此阶段，前 100 次有效温度样本（耗时约 150 秒）被送入在线均值迭代更新方程中计算 Baseline 温度基准线 $\mu_{base}$：
$$\mu_{base}[n] = \frac{n-1}{n}\mu_{base}[n-1] + \frac{1}{n}T_f[n]$$
更新 100 次后，$\mu_{base}$ 被锁定，状态转为监测模式（\texttt{AI\_STATE\_NORMAL}）。系统周期性监视温偏差 $\Delta T = |T_f - \mu_{base}|$：
- 当 $\Delta T \le 5^\circ\text{C}$ 时：表示环境温度处于安全区间，绿色指示灯呈现 10ms 级别的平滑呼吸效果。
- 当 $\Delta T > 5^\circ\text{C}$ 时：表示环境温度发生异常突变，状态机跃迁为报警状态（\texttt{AI\_STATE\_\allowbreak ANOMALY}），切断呼吸灯并控制红色指示灯以 5Hz 频率高频闪烁（蜂鸣器关闭以防止惊扰环境），并记录报警事件快照存入容量为 5 的异常环形日志缓冲区（Ring Buffer）中。

\subsection{I2C 总线热插拔自愈算法 (Self-Healing)}
针对微机实验中由于导线晃动极易引发的总线掉线问题，系统引入了自愈机制。每 1.5 秒主循环读取一次数据，若连续 3 次遇到 I2C 状态超时或 NACK 丢失，系统自动标记传感器为 `OFFLINE`，避免长时间卡死在通信等待逻辑中。在此状态下，主循环在后台每个 1.5 秒周期调用一次初始化，一旦传感器物理重新连入总线，系统可在 1.5 秒窗口内实现链路完全自愈唤醒。

\section{交互层设计与增量游戏引擎实现}

\subsection{暗 Slate 霓虹 / Claude 暖白双主题调度}
人机交互界面集成于 480×320 像素的 ST7796S 液晶屏上，支持由 BTN2 按键触发的主题快速反转重绘：
- **Slate 霓虹暗黑主题**：使用纯黑背景（`0x0000`）辅以 Slate 深灰卡片（`0x2104`），搭配高亮青色与绿色文本，在暗光下展现极强科技质感。
- **Claude Paper 亮色主题**：背景采用温暖米黄纸张色（`0xF7BD`），卡片为白，文字为深褐，在强日光下具备极佳视认度。

\subsection{示波器风格网格折线图}
在 Page 1 AI 基准页与 Page 4 防盗安全页中，右侧各内置了一个 $248 \times 220$ 像素的绘图看板。图表背景由 \texttt{Draw\_DottedLine} 函数绘制出的点阵虚线组成示波网格（如 50cm，100cm 等刻度线）。后台维护过去 24 次的偏差/距离数据，基于当前坐标系尺度自适应缩放后，使用 Bresenham 划线算法进行点对点高速像素连接，描绘出一条高视觉质感的实时趋势波形。

\subsection{慢速 SPI 带宽下 “脏矩形” 局部增量刷新技术}
在 Page 3 的像素小游戏 Flappy Bird 中，动作更新的心跳节拍为 30ms。如果采用传统全屏刷屏重画方案，每帧需要传输 $480 \times 320 \times 2\text{ 字节} = 307.2\text{ KB}$ 的显存数据。在软件模拟 SPI 约 1.25 Mbps 的实际有效速率下，传输全屏像素数据需耗费：
$$\text{Time}_{full} = \frac{307.2 \times 1024 \times 8}{1.25 \times 10^6} \approx 2013\text{ ms} = 2.01\text{ 秒}$$
这会使得帧率低于 0.5 FPS，产生严重的画面闪烁。

为了打破总线时宽瓶颈，项目研发了 **脏矩形（Dirty Rects）局部增量刷新算法**。其基本原理是每一帧重绘时仅修改发生变化的像素区域：
- **小鸟增量擦除与重绘**：小鸟物理大小为 12×12 像素。每一帧游戏首先用背景色填充小鸟在上一帧的旧坐标位置，以抹除其飞行残影；然后在新坐标写入小鸟颜色（每次传输仅需 $12 \times 12 \times 2 \times 2 \approx 576\text{ 字节}$）。
- **管道增量移动绘制**：管道每次向左平移 4 像素。渲染时无需重绘 32 像素宽的管道卡片，而是仅仅用背景色填充管道右侧由于平移暴露出的 $4 \times 230$ 像素窄条（擦除），并用绿色在新管道左侧补画 $4 \times 230$ 像素窄条（重绘）。

通过脏矩形局部增量刷新，单帧的总线像素数据传输量急剧下降为：
$$\text{DataSize}_{dirty} = (12 \times 12 \times 2) + (4 \times 230 \times 2 \times 2) \approx 3.96\text{ KB}$$
总线总耗时由 2.01 秒缩短至：
$$\text{Time}_{dirty} = \frac{3.96 \times 1024 \times 8}{1.25 \times 10^6} \approx 25.9\text{ ms}$$
传输数据量降低了 \textbf{98.7\%}，这使得动作游戏可以完美适配 30ms 游戏循环，实现了稳定在 \textbf{30+ FPS} 的无闪烁流畅动作画面。

\section{Git 双分支模块化系统展示设计}
受限于 STM32F103C8T6 紧张的引脚资源，液晶屏 XPT2046 触控 SPI（\texttt{TCS}、\texttt{TCLK}、\texttt{TDIN}、\texttt{PEN}）与 SD卡的 SPI（\texttt{SD\_CS}、\texttt{SD\_SCK}、\texttt{SD\_MOSI}、\texttt{SD\_MISO}）存在物理重叠冲突。同时在 \texttt{PEN} 触摸引脚拉低时会强制牵拉 SD卡的 \texttt{MISO} 数据线，存在总线电气短路和干扰的风险。

为解决这一硬件瓶颈，项目引入了 Git 双分支管理架构。在保持面包板硬件连线完全不动的前提下，分固件进行编译与展示：
1. \textbf{IoT 综合分支 (\texttt{master})}：屏蔽触摸屏驱动，启用 SD卡 SPI。在此固件下，系统专注于多传感器采集、网络 NTP 北京时钟同步、心知天气广州数据拉取、微信 Server酱 实时云端报警推送、数码管温度显示、超声波警戒布防以及 SD卡 绝对时间日志连续存储。
2. \textbf{触控交互小游戏分支 (\texttt{feat-touch})}：屏蔽 SD卡 控制线，使能 XPT2046 触摸芯片。在此固件下，系统提供丰富的触屏人机交互：用户可通过轻触屏幕最右侧的小主题矩形框直接切换主题，在 Page 1 触点点击 \texttt{[RST]} 重置 AI 学习，在 Page 4 防盗警戒页触点点击 \texttt{[+]} 和 \texttt{[-]} 微调报警距离阈值，以及在 Page 3 屏幕任意区域触屏控制小鸟起飞游戏。

两套固件在编译阶段均确保 \textbf{0 错误、0 警告}。展示时只需利用 Keil uVision 分别烧录即可完整展示全部大作业特色指标，具有很强的工程复用度。
